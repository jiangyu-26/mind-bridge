# 心理 AI 助手前端项目总结

## 项目名称

心理 AI 助手前端系统

## 项目描述

基于 Vue 3 + Vite 搭建的心理健康 AI 助手平台，面向用户提供 AI 心理咨询、情绪日记、情绪分析结果展示、心理知识库阅读等功能，同时提供后台管理端用于知识文章、咨询记录、情绪日志和数据分析管理。项目通过 Axios 对接后端 RESTful API，并使用 SSE 实现 AI 咨询流式响应，提升对话交互体验。

## 技术栈

- 前端框架：Vue 3、Composition API、Vite
- UI 组件库：Element Plus、Element Plus Icons
- 路由与状态：Vue Router、Pinia
- 网络请求：Axios、请求/响应拦截器、Token 鉴权
- 数据可视化：ECharts
- 实时通信：Server-Sent Events，基于 `@microsoft/fetch-event-source`
- 内容渲染：Markdown 渲染组件、富文本编辑器 WangEditor
- 样式方案：SCSS、响应式布局、组件级 scoped 样式

## 核心功能

### 用户端

- 登录注册与路由鉴权，根据 Token 控制用户端和后台管理端访问权限。
- AI 心理咨询模块，支持新建会话、历史会话切换、会话删除、消息流式输出和异常状态展示。
- 情绪分析面板，接入 emotion 接口渲染情绪评分、主导情绪、治愈行动、风险提示等 AI 分析结果。
- 情绪日记模块，支持 1-10 分情绪评分、情绪标签选择、触发因素记录、日记内容填写、睡眠质量和压力水平录入，并对接创建/更新情绪日记接口。
- 心理知识库模块，支持推荐阅读、文章分页列表、文章详情查看、Markdown 正文渲染和标签展示。

### 后台管理端

- 数据看板模块，使用 ECharts 展示用户、情绪日志、咨询记录等趋势数据。
- 知识文章管理模块，支持文章分页查询、新增、编辑、发布、下线、删除、封面上传和富文本编辑。
- 咨询记录管理模块，支持咨询会话列表查看和消息详情查看。
- 情绪日志管理模块，支持情绪日记分页查询、详情查看、AI 分析结果展示和记录删除。

## 技术亮点

- 使用 `fetchEventSource` 实现 AI 回复流式渲染，相比普通接口返回降低用户等待感，并补充连接失败、服务异常、流结束等状态处理。
- 抽离 emotion 数据归一化工具，统一处理 `riskLevel`、`riskDescription`、`improvementSuggestions` 等字段，提升接口字段变化时的容错能力。
- 将复杂页面按职责拆分为多个可复用组件，父组件只保留接口调用和状态流转，降低单文件复杂度。
- 封装 Axios 响应拦截器，统一处理业务状态码、错误提示和 Token 注入，减少页面内重复代码。
- 使用 ECharts 渲染后台趋势数据，并通过 computed 对后端多种可能字段进行兼容处理。
- 使用 Element Plus 表单、表格、分页、弹窗、上传等组件快速构建管理端，提高开发效率和交互一致性。

## 面试可展开说明

- AI 咨询为什么使用 SSE：适合服务端持续推送文本片段，能做到边生成边展示，比等待完整响应体验更好。
- 情绪分析接口如何容错：前端对接口字段做统一归一化，兼容数组、字符串、JSON 字符串和不同命名字段。
- 页面如何降低复杂度：将业务状态留在页面容器，把展示区域拆成纯 UI 组件，通过 props 和 emit 传递数据与事件。
- 如何处理鉴权：登录后保存 Token，Axios 请求拦截器统一注入请求头，路由守卫控制需要登录的页面访问。
